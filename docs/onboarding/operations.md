# Operations Onboarding Pack

This guide is for branch operations staff, admins, and managing-partner support users responsible for intake coordination and workflow movement.

## First-Day Setup

- confirm login works on web
- confirm the correct role and branch access are assigned
- open:
  - dashboard
  - team
  - clients
  - properties
  - cases
  - documents
- review the branch filter behavior if your role can access multiple branches
- review the notifications panel and audit page

## First-Week Learning Path

### Day 1

- read the [operations SOP](/Users/michaeladeleye/Documents/product/ERP Surveyors/docs/sops/operations.md)
- review the core workflow in [docs/current-functionality-bible.md](/Users/michaeladeleye/Documents/product/ERP Surveyors/docs/current-functionality-bible.md)
- explore seeded demo records from client through invoice

### Day 2

- create a test client
- create a test property
- create a test case with branch, valuer, reviewer, and due date
- review the case activity feed and checklist

### Day 3

- practice reassigning a case
- practice using saved views and filters on clients/cases
- review overdue and review-stage lists

### Day 4

- upload and link supporting documents
- confirm document linkage rules across client/property/case
- review notification behavior after reassignment

### Day 5

- run a full intake-to-ready-for-inspection walkthrough
- shadow a real or demo case handoff to a valuer

## Role Readiness Checklist

The team lead should confirm the person can:

- create and update client records correctly
- create and update property records correctly
- open a case with the right branch and ownership
- identify when a stage is blocked
- use documents without leaving orphaned uploads
- recognize overdue and at-risk items on the dashboard
- escalate issues to the right team

## Daily Habits

- review dashboard summary at the start of day
- check notifications for assignments and overdue changes
- keep new intake records clean before cases are opened
- review review-stage and overdue queues before end of day

## Weekly Habits

- review stuck cases by stage
- review unassigned or ambiguous branch ownership records
- review document completeness on active cases
- review team workload distribution with management

## KPIs To Watch

- average time from client/property readiness to case opened
- number of overdue active cases by branch
- number of cases stuck in the same stage beyond SLA
- number of cases missing clear valuer or reviewer ownership
- document completeness rate on active cases

## 30-Day Signoff Questions

- can the person independently open and assign a case correctly?
- can they identify workflow risk before it becomes overdue?
- do they understand who owns the next handoff at each stage?
- are they using notifications, audit, and filters effectively?
