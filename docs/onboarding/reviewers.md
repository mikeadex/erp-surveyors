# Reviewer Onboarding Pack

This guide is for senior reviewers responsible for quality control, comment discipline, approval, and issue readiness.

## First-Day Setup

- confirm web login works
- open:
  - dashboard
  - cases
  - reports
  - audit
  - notifications
- review report comment history and approval flow on a seeded case

## First-Week Learning Path

### Day 1

- read the [reviewer SOP](/Users/michaeladeleye/Documents/product/ERP Surveyors/docs/sops/reviewers.md)
- review how reports are generated from analysis data
- understand readiness and compliance gates

### Day 2

- open a seeded report under review
- inspect the report preview, comments, and case context
- compare report content with inspection and analysis inputs

### Day 3

- add blocking, suggestion, and informational comments in a demo flow
- review how comment resolution behaves

### Day 4

- practice approval and rejection decisions
- review audit history for review actions

### Day 5

- review two or three seeded cases end to end and document acceptance reasoning

## Role Readiness Checklist

The team lead should confirm the person can:

- distinguish blocking issues from suggestions
- assess whether a report conclusion is evidence-backed
- use the comment workflow correctly
- approve only when readiness gates are genuinely met
- identify compliance or consistency problems quickly

## Daily Habits

- review submitted reports promptly
- prioritize blocking issues first
- keep comments specific and actionable
- avoid approval without confirming the underlying workbench and inspection state

## Weekly Habits

- review common rejection/comment themes with valuers
- review audit history for process drift
- review turnaround time on submitted reports

## KPIs To Watch

- average review turnaround time
- percentage of reports approved on first review
- average blocking comments per submitted report
- unresolved comment age
- rejection rate by valuer or branch

## 30-Day Signoff Questions

- can the person review a report independently and correctly?
- are their comments clear enough for valuers to act on?
- do they apply approval standards consistently?
- are they using audit and readiness cues effectively?
