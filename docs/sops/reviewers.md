# Reviewer SOP

This SOP is for senior reviewers responsible for quality control, comment resolution discipline, and final approval before issue.

## Primary Objective

Ensure that every report approved for issue is evidence-backed, internally consistent, and compliant with the firm’s valuation standards.

## Main Areas You Own

- review-stage case assessment
- report comment creation
- approval or rejection decisions
- compliance and readiness checks

## Daily Workflow

1. Review dashboard and notifications for reports pending review.
2. Open each submitted report and confirm:
   - report content is present
   - inspection evidence exists
   - comparables are attached
   - analysis is complete
3. Read the report narrative and compare it with:
   - inspection findings
   - analysis summary
   - comparable grid
   - reconciliation note
4. Add review comments where clarification, correction, or caution is required.
5. Use approval only when the report is ready for issue.
6. Use rejection when the report needs meaningful rework rather than minor cleanup.

## Comment Types

- `blocking`
  use when the report should not progress until fixed
- `suggestion`
  use for improvement that does not invalidate the report
- `informational`
  use for context or lightweight reviewer notes

## Approval Checklist

Before approving, confirm:

- no blocking issue remains
- the value conclusion is supported by the analysis
- assumptions are appropriate and not contradictory
- the report reflects the actual property and client record
- inspection coverage is sufficient
- the comparable evidence is reasonable for the assignment

## Resolution Rules

- only the assigned valuer or managing partner should resolve review comments
- unresolved blocking comments should prevent final progression
- approvals and issue events should happen in-system so audit history remains complete

## Escalation Triggers

Escalate when:

- the evidence is clearly insufficient
- the conclusion is not supported by the comparable logic
- the report uses the wrong property/client context
- a report is repeatedly resubmitted without addressing major blockers

## Handoffs

- from valuers:
  submitted draft reports
- back to valuers:
  review comments and rejection decisions
- to management:
  approved reports ready for final issue where required
