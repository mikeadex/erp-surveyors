# Valuer SOP

This SOP is for valuers and surveyors who own the valuation workflow from case execution through analysis and draft reporting.

## Primary Objective

Turn an assigned case into a defensible valuation conclusion supported by evidence, analysis, and a review-ready report.

## Main Areas You Own

- case execution
- inspection coordination or participation
- comparable selection and case-specific adjustments
- valuation analysis and assumptions
- draft report preparation
- response to reviewer comments

## Daily Workflow

1. Review your dashboard and assigned case list.
2. Prioritize:
   - overdue cases
   - inspection-dependent cases
   - cases already in analysis or draft report stages
3. Open the case record and review:
   - client context
   - property context
   - inspection status
   - documents
   - prior activity
4. Make sure the inspection has been submitted before moving into final analysis.
5. Attach comparables from the library or capture missing evidence.
6. Complete the valuation workbench:
   - assumptions
   - special assumptions
   - weighting
   - adjustments
   - concluded value
   - reconciliation commentary
7. Generate or revise the report draft.
8. Submit the report for review when it is ready.

## Comparable Rules

When using comparables:

- prefer verified evidence where possible
- record weight, relevance, and adjustment per case
- use the same library comparable across multiple cases only when it is genuinely relevant
- explain major divergence between weighted indication and concluded value in the reconciliation note

## Analysis Checklist

Before saving the final workbench state, confirm:

- the conclusion basis is set
- assumptions and special assumptions are complete
- comparables are attached and adjusted meaningfully
- the comparable grid reflects the actual evidence logic
- concluded value is filled in
- commentary explains the judgment, not just the numbers

## Report Readiness Checklist

Before submitting for review, confirm:

- the inspection has been submitted
- comparable evidence is attached
- analysis is complete
- the report narrative reads clearly
- the valuation conclusion matches the workbench
- supporting documents are linked where relevant

## Review Response Rules

When a reviewer comments:

- treat blocking comments first
- resolve comments inside the system so history stays visible
- update analysis/report content before marking a point resolved
- do not issue a report directly if it still has unresolved blockers

## Handoffs

- from operations:
  a complete, assigned case
- to reviewers:
  a report in `submitted_for_review`
- to finance:
  completed work after final issue
